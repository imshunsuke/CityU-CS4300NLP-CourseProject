# Smart Meeting Assistant

CS4300 NLP group project · Topic 5.
Bilingual (Chinese–English) meeting assistant with six views: transcript, summary, translation, action items, sentiment timeline, and retrieval-augmented Q&A — plus a live microphone mode for real-time S2T + translation.

## Setup

```bash
# 1. Conda env (brings its own ffmpeg — required by WhisperX)
conda create -n sma python=3.10 ffmpeg -c conda-forge -y
conda activate sma
pip install -r requirements.txt

# 2. Secrets
cp .env.example .env
# edit .env:
#   OPENAI_API_KEY=sk-...
#   OPENAI_BASE_URL=...      (optional, for proxies)
#   HF_TOKEN=hf_...
```

**Then click "Agree" on these three gated HuggingFace pages** (one-time, with the account behind your HF token). WhisperX will silently fail otherwise.
- https://hf.co/pyannote/speaker-diarization-3.1
- https://hf.co/pyannote/segmentation-3.0
- https://hf.co/pyannote/speaker-diarization-community-1

## Run

```bash
conda activate sma
python app.py    # → http://localhost:7860
```

Two tabs:
- **Batch analysis** — upload an audio file, explore six sub-tabs (transcript / summary / translation / actions / sentiment / Q&A). First run ~90 s for a 2-min clip; results are cached by audio hash.
- **Live microphone** — click mic, speak a sentence, pause. Transcript appears in ~1-2 s, translation fills in ~1-2 s later. Click **Process now** to force-flush.

Or run the CLI: `python -m src.pipeline --audio <file.wav> --target-lang zh`.

## Data + evaluation

See [`eval/`](eval) for metric scripts. Manifests are generated by `eval/data_prep/*.py`. Download instructions for AMI / AliMeeting / MELD are in the report appendix.

## Gotchas

- Use `conda activate sma` (or `conda run -n sma`) when invoking `python` — otherwise `ffmpeg` isn't on PATH and WhisperX dies.
- If `wav2vec2_fairseq_base_ls960_asr_ls960.pth` download errors with `PytorchStreamReader failed`, `curl --retry 5 -C -` it manually into `~/.cache/torch/hub/checkpoints/`.
- `sentence-transformers` must be `<4`; 5.x hard-imports torchcodec which doesn't load against conda's ffmpeg 8.
- WhisperX ASR runs on CPU (ctranslate2 has no MPS support). Alignment + diarization run on MPS automatically.
